# ADR 003: Сессии через cookie + таблица Session

## Статус

Принято (`src/server/auth/session.ts`).

## Контекст

Нужна простая серверная авторизация без внешнего IdP для небольшой команды.

## Решение

- Сессия хранится в БД (**`Session`**), в браузере — httpOnly cookie **`wowstorg_session`**.
- В production cookie с флагом **`secure`**.

## Последствия

- Масштабирование на несколько регионов Vercel без sticky sessions может потребовать общего хранилища сессий — пока не требуется.
- Не смешивать с JWT в localStorage без миграционного плана.

## Нарушать?

Только через ADR и миграцию пользователей/сессий.
