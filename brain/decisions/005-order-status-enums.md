# ADR 005: Статусы заказов и сущностей — только Prisma enum

## Статус

Принято (схема `OrderStatus`, `OrderSource`, и др.).

## Контекст

Строковые статусы без схемы приводят к рассинхрону UI, API и БД.

## Решение

Переходы и хранение статусов заказа — через **enum в `schema.prisma`**. Новые состояния — только через миграцию и обновление всех switch/графов в UI.

## Последствия

- Любой новый статус — большая задача (БД + API + UI + уведомления).
- Временные «костыли» через произвольные строки в JSON запрещены для статуса заказа.

## Нарушать?

Нет; для новых сущностей по возможности сразу enum.
