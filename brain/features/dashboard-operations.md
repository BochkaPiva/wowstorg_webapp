# Операционный дашборд

## Цель

Пересобрать Wowstorg-дэшборд на `/home` из сводки по заявкам в рабочий экран на ближайшие дни: задачи, проекты, заявки и складские риски должны давать понятный план действий без визуального шума.

## Границы

- Не менять схему БД в первой итерации.
- Не удалять текущие заявки, склад и календарь выдачи, а переставить их по приоритету.
- Не показывать сигналы по просто незаполненным полям без срока или риска.
- Не делать новую систему статусов проектов/заявок.
- Не превращать дашборд в аналитику: годовой heatmap по умолчанию свернут.

## Затронуто

- UI: `/home`, `src/app/home/page.tsx`, `src/app/home/IssuanceCalendar.tsx`.
- API: новый или расширенный Wowstorg dashboard API.
- Server: новый агрегатор `src/server/dashboard/operations.ts`.
- Existing data: `Order`, `Project`, `WorkTask`, `WorkTaskColumn`, `ProjectNotificationCooldown`, складские поля `Item`.

## Инварианты

- Первый экран отвечает на вопрос «что делать сегодня и скоро».
- Сигнал появляется только при риске срока, денег или операционного срыва.
- У каждого проектного сигнала есть понятная причина и возможность отложить, если это не критика на сегодня.
- Дедлайны задач трактуются как date-only по текущему паттерну проекта.
- Закрытые/отмененные проекты и скрытые задачи не должны попадать в основной дашборд.

## Сигналы MVP

- `TASK_OVERDUE`: незавершенная задача с дедлайном раньше сегодня.
- `TASK_DUE_SOON`: моя незавершенная задача с дедлайном сегодня/завтра.
- `PROJECT_EVENT_SOON_WITHOUT_ESTIMATE`: активный проект с подтвержденной датой в ближайшие 14 дней без основной сметы.
- `PROJECT_BLOCKED`: активный проект с заполненным `openBlockers`.
- `ORDER_STALE`: активная заявка рядом по датам, которая не обновлялась 3+ дня.

Опционально после MVP:
- `PROJECT_EVENT_SOON_WITHOUT_OPEN_TASKS`.
- `WAREHOUSE_SHORTAGE_SOON`.

## Приемка

- [ ] На `/home` есть компактные блоки `Сегодня`, `Ближайшие дни`, `Мои задачи`, `Сигналы`.
- [ ] Календарь выдачи свернут по умолчанию и раскрывается вручную.
- [ ] Блоки не содержат очевидных описательных подзаголовков.
- [ ] Сигналы соответствуют правилам из `docs/DASHBOARD_OPERATIONS_REDESIGN_PLAN.md`.
- [ ] Текущие заявки и склад доступны ниже первого экрана.
- [ ] `npm run build` проходит.
