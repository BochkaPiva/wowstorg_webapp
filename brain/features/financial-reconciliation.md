# Финансовая сверка Excel ↔ WebApp

## Цель

Показывать расхождения между внешней финансовой таблицей и фактом WebApp, не подменяя исходные заявки, проекты, сметы и складские операции.

## Границы

- Загрузка принимает только `.xlsx` и `.xls`.
- Первая операция всегда формирует предварительный просмотр без записи.
- Сохранение создаёт неизменяемый снимок внешнего источника и строки сопоставления.
- Сверка не обновляет финансы `Order`, `Project` или `ProjectEstimateVersion`.
- Неоднозначные ссылки и названия получают статус `CONFLICT` и требуют ручного решения.

## Затронуто

- Prisma: `FinancialReconciliationBatch`, `FinancialReconciliationRow`, `FinancialReconciliationMatchStatus`.
- API: `GET/POST /api/admin/analytics/reconciliation`.
- UI: вкладка «Сверка» на `/admin/analytics`.
- Парсер: `src/server/analytics/reconciliation.ts`.

## Инварианты

- Снимок хранит значения исходного Excel отдельно от оперативных данных.
- Сопоставление по ссылке имеет приоритет над точным названием.
- Несовпадение названия сущности и строки Excel не считается точным совпадением.
- Период бонусов выбирается независимо от периодов других аналитических разделов.
- Таблицы снимков недоступны ролям `anon` и `authenticated` через Supabase Data API.

## Приёмка

- [x] Excel можно проверить без записи.
- [x] После подтверждения снимок появляется в истории.
- [x] Видны итоги Excel, факт сайта и дельта.
- [x] Видны совпавшие, конфликтные и несопоставленные строки.
- [x] Денежные значения распознаются в российских и смешанных Excel-форматах.
- [x] Исходные финансовые сущности не изменяются.
