# Миниатюры товаров в заявках

## Цель
Сделать основной флоу заявок нагляднее: показывать маленькие фото товаров в составе заявки, при редактировании и на приемке, не меняя бизнес-логику заказов.

## Границы (out of scope)
- Не трогаем status machine заявок.
- Не меняем транзакционные маршруты `warehouse-edit`, `greenwich-edit`, `return-declared`, `check-in`.
- Не добавляем новые media API, buckets или stored thumbnails.
- Не расширяем в этой итерации списки заявок, очередь склада, ремонт, утери, `in-rent` и проектную смету.

## Затронуто
- API: `src/app/api/orders/[id]/route.ts`
- UI: `src/app/orders/[id]/page.tsx`
- Фото: `src/app/api/inventory/positions/[id]/photo/route.ts` (переиспользование существующего preview-механизма)
- Документация: `docs/ORDER_THUMBNAILS_ROLLOUT.md`

## Инварианты
- Используем только существующий маршрут фото `/api/inventory/positions/[id]/photo?w=...`.
- На экране заявки рендерим `<img>` только если у позиции есть `photo1Key`, чтобы не плодить лишние `404`.
- Превью только маленькие, с `loading="lazy"` и `decoding="async"`.
- Layout таблиц и карточек приемки должен остаться стабильным и читабельным на узких экранах.
- Embed-режим заявки должен автоматически получить те же миниатюры без отдельной ветки логики.

## Приемка
- [ ] В read-only составе заявки есть миниатюры товаров.
- [ ] В режиме редактирования строки заявки показывают миниатюры.
- [ ] В `Добавить позицию` миниатюры видны в поисковой выдаче и в выбранной позиции.
- [ ] В модалке Greenwich и в складской приемке миниатюры не ломают форму и скролл.
- [ ] При отсутствии фото показывается аккуратный placeholder.
- [ ] Бизнес-логика приемки и редактирования не меняется.
