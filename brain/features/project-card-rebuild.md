# Пересборка карточки проекта

## Цель

Сделать карточку проекта рабочим центром: быстро видно состояние проекта, чей ход, блокеры, задачи, заявки реквизита и финансовую картину без дублирования полей и лишних переходов.

## Границы

- В первой волне не меняем бизнес-модель проектов и смет без отдельного решения.
- Не создаём отдельную систему проектных задач: используем существующие `WorkTask` с `projectId`.
- Не удаляем степпер заявок реквизита.
- Не меняем формулы сметы и XLSX/PDF-контракты при визуальном редизайне.

## Затронуто

- Prisma: без обязательной миграции; используются существующие `Project`, `Order.projectId`, `WorkTask.projectId`, `WorkTask.orderId`.
- API: `/api/projects/[id]`, `/api/tasks/boards/[id]`, `/api/tasks/**`.
- UI: `/projects/[id]`, `/tasks`, `ProjectEstimatePanel`.
- Документация: подробный план в `docs/PROJECT_CARD_REBUILD_PLAN.md`.

## Инварианты

- Архивный проект остаётся read-only.
- Задачи закрытого/архивного проекта не должны шуметь в основной доске, но не удаляются.
- Проектный YouGile и основной YouGile показывают одни и те же задачи через разные фильтры.
- Заявки реквизита остаются связанными с `Order`, а сметные блоки реквизита остаются 1:1 к заявкам.
- Сметная математика, комиссия и налог не меняются в рамках UI-пересборки.

## Приёмка

- [ ] В шапке редактируются название, даты, статус и мяч.
- [ ] Блок "Карточка проекта" как дубль шапки удалён.
- [ ] Статус меняется через рекомендуемое следующее действие и сгруппированное меню.
- [ ] Мяч меняется через компактный сегментированный контроль.
- [ ] Рабочие заметки компактны и не занимают большой пустой блок.
- [ ] В проекте есть блок задач, показывающий задачи с `projectId` текущего проекта.
- [ ] Задачи синхронно видны в проекте и основном YouGile.
- [ ] Задачи закрытых/архивных проектов скрыты из основной доски по умолчанию.
- [ ] Степпер заявок реквизита сохранён.
- [ ] Финансовые итоги сметы сгруппированы по смыслу: клиент, внутреннее, маржа.
