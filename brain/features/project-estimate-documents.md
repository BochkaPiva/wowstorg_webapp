# Несколько смет в проекте

## Цель

Заменить бизнес-логику "версий одной сметы" на несколько самостоятельных смет внутри проекта. Каждая смета имеет ручное название, свои заявки реквизита, ручные разделы и собственные итоги; проект суммирует учитываемые сметы.

## Границы

- Статусы смет не вводим.
- Историю старых редакций не храним как версии в базе; при необходимости пользователь сохраняет XLSX-файл.
- Одна заявка проекта может быть добавлена только в одну смету.
- Генератор доп. соглашений не входит в эту итерацию.

## Затронуто

- Prisma: новая сущность сметы проекта, перенос связей секций сметы с версии на смету.
- API: сметы проекта, добавление/удаление заявок из сметы, XLSX по выбранной смете.
- UI: `ProjectEstimatePanel`, карточка проекта, модалка выбора заявок.
- Аналитика: список проектов, админская финансовая аналитика, финансовый XLSX-отчёт.

## Инварианты

- Новая смета создаётся пустой.
- Заявки добавляются в смету только вручную.
- Заявку нельзя добавить в две сметы одного проекта.
- Удаление заявки из сметы не удаляет саму заявку.
- Ручные разделы считаются только в той смете, где созданы.
- Финансы проекта = сумма смет с включённым флагом "Учитывать в итогах проекта".
- Заявки с `projectId` не участвуют в самостоятельной финансовой аналитике заявок.
- Нераспределённые заявки проекта должны подсвечиваться сигналом.

## Приёмка

- [ ] В проекте можно создать несколько смет с ручными названиями.
- [ ] В смету можно добавить заявки через модалку с чекбоксами.
- [ ] Уже распределённые заявки видны в модалке как недоступные с указанием сметы.
- [ ] Итог отдельной сметы и итог проекта считаются без двойного учёта.
- [ ] XLSX выгружается по конкретной смете.
- [ ] Существующие проекты после миграции получают одну смету "Смета проекта".
