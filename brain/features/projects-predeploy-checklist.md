# Проекты: что сделать вручную перед/после выката

Краткий список инфраструктуры, которую нельзя зафиксировать только кодом.

## Supabase (Preview и Production)

1. Создать bucket **`project-files`** (или имя из `SUPABASE_STORAGE_PROJECTS_BUCKET`), политики доступа в духе остальных файловых бакетов приложения.
2. Убедиться, что сервисный ключ / RLS настроены так же, как для существующих бакетов.

## Vercel

1. Добавить переменную **`SUPABASE_STORAGE_PROJECTS_BUCKET`** для Preview и Production (значения должны соответствовать бакетам в соответствующих проектах Supabase).
2. После добавления переменных — **пересобрать** деплой.

## База

1. На каждой среде выполнить **`npm run db:deploy`** (или эквивалент CI), затем при необходимости сид.

Загрузки файлов проекта **уже в коде** — без бакета и env они не заработают в production. Полный чеклист после реализации и ручные проверки: [`projects-post-implementation-checklist.md`](./projects-post-implementation-checklist.md). Список API и env синхронизируйте через `npm run brain:inventory`.
