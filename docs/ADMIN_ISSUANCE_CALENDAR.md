# Календарь выдачи реквизита (дашборд на главной)

## Назначение

На **главной** (`/home`) в блоке **«Дашборд»** (роль Wowstorg) показывается **heatmap за год** (сетка «как на GitHub»): закрашиваются дни, в которые по заявкам шла **выдача реквизита** — то есть дни периода **`startDate`…`endDate`** (включительно).

- Учитываются **все заявки**, кроме со статусом **`CANCELLED`** (включая доп.-выдачи с `parentOrderId`).
- Если в один день попадает несколько заявок (или периоды пересекаются), **интенсивность цвета выше**.

## Данные и API

- **GET** `/api/dashboard/issuance-calendar?year=YYYY`  
  - Доступ: только роль **`WOWSTORG`**.  
  - Ответ: `{ year, days: { "YYYY-MM-DD": number }, maxCount, orderCount }`  
  - Счётчики считаются в **UTC** по датам из БД (date-only, полуночь UTC), как и остальной код про заявки.

## UI

- Компонент: `src/app/home/IssuanceCalendar.tsx`
- Сетка **на всю ширину** блока: недели делят ширину поровну (`flex-1`), квадраты — `aspect-square`.
- На **узких экранах** (`max-sm`) задана минимальная ширина графика (~640px), включена **горизонтальная прокрутка** и подсказка «листайте вправо» — иначе клетки были бы слишком мелкими.
- Недели — **столбцы**, сверху вниз дни **вс → сб** (как на GitHub).
- Выбор года: последние 6 лет включая текущий.

## Ограничения и развитие

- Нет отдельного кэша: при большом числе заявок пересчёт на каждый запрос остаётся лёгким для типичных объёмов; при необходимости можно добавить агрегат по дням или кэш.
- Семантика «выдача» здесь = **календарный период аренды в заявке**, а не фактический timestamp нажатия «Выдать» на складе.
