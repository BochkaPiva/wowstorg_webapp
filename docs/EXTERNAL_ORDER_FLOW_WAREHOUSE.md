# Внешние заявки склада (`WOWSTORG_EXTERNAL`): разбор и план исправления

## Контекст

Проблема: заявки, которые склад создаёт для внешних клиентов (`source=WOWSTORG_EXTERNAL`), проходят тот же сценарий, что и внутренние (`GREENWICH_INTERNAL`), из-за чего флоу ожидает действия Greenwich там, где участника Greenwich нет.

Нужно:
- сразу переводить внешнюю заявку в этап «Согласована»;
- формировать смету сразу (файл доступен в карточке заявки сразу после создания);
- сохранять редактирование складом до старта сборки;
- не требовать участия Greenwich в шагах, где его по бизнесу нет.

## Текущее состояние (до правок)

### Создание заявки
- Роут: `POST /api/orders` (`src/app/api/orders/route.ts`).
- Независимо от `source` заявка создаётся со статусом `SUBMITTED`.
- Для внешней заявки это создаёт лишний шаг «отправить смету / ждать Greenwich».

### Отправка сметы
- Роут: `POST /api/orders/:id/send-estimate`.
- Разрешён только из `SUBMITTED` и `CHANGES_REQUESTED`.
- Ставит статус `ESTIMATE_SENT` (или `APPROVED_BY_GREENWICH` только в особом кейсе повторной отправки без изменений).

### Редактирование складом
- Роут: `PATCH /api/orders/:id/warehouse-edit`.
- Разрешает правки в `SUBMITTED`, `ESTIMATE_SENT`, `CHANGES_REQUESTED`, `APPROVED_BY_GREENWICH`, `PICKING`.
- Если статус был `APPROVED_BY_GREENWICH`/`PICKING`, откатывает статус в `SUBMITTED`.
- Для внешних заявок это возвращает их в Greenwich-зависимую ветку.

### Старт сборки
- Роут: `POST /api/orders/:id/start-picking`.
- Требует статус `APPROVED_BY_GREENWICH`.
- Само требование корректно, но текст ошибки ссылается на Greenwich как на обязательного участника.

### Нотификации
- `notifyEstimateSent`, `notifyStartPicking`, `notifyIssued`, и др. ориентированы на рассылку в Greenwich ЛС.
- Для внешних заявок это нецелевой канал.

## Целевое поведение

### Для `source=WOWSTORG_EXTERNAL`
1. При создании:
   - проверка цен включённых услуг (если услуга включена, цена обязательна > 0);
   - формирование xlsx сметы;
   - запись `estimateFileKey`, snapshots;
   - статус сразу `APPROVED_BY_GREENWICH`.
2. При редактировании складом (до `PICKING`):
   - изменения сохраняются;
   - статус остаётся `APPROVED_BY_GREENWICH` (не откатывается в `SUBMITTED`);
   - смета пересобирается и обновляется.
3. Старт сборки:
   - как и раньше, из `APPROVED_BY_GREENWICH`.
4. Никаких обязательных действий Greenwich в этой ветке.
5. После выдачи внешний заказ может быть отправлен на приёмку самим складом (без Greenwich).

### Для `source=GREENWICH_INTERNAL`
- Текущая логика сохраняется без изменений.

## Технический план правок

1. Добавить серверный helper для сметы (переиспользуемый):
   - валидация цен включённых услуг;
   - сбор snapshot строк;
   - генерация xlsx;
   - запись файла в `data/estimates`.
2. Обновить `POST /api/orders`:
   - ветка для `WOWSTORG_EXTERNAL` делает auto-approve + auto-estimate.
3. Обновить `PATCH /api/orders/:id/warehouse-edit`:
   - для внешних заявок не делать reset в `SUBMITTED`;
   - после изменения пересобирать смету и сохранять файл/снимки.
4. Обновить `POST /api/orders/:id/start-picking`:
   - нейтральный текст ошибки (без жёсткой привязки к Greenwich).
5. Обновить UI-условия редактирования:
   - редактирование только до `PICKING` (в терминах задачи: «до начала сборки»).
6. Обновить возврат для внешних заявок:
   - `POST /return-declared` разрешить складу, если у заявки нет `greenwichUserId`.
   - добавить кнопку «На приёмку» для склада на этапе `ISSUED` у внешней заявки.

## Риски и их покрытие

1. **Поломка внутреннего Greenwich-флоу**  
   Покрытие: логика ветвится строго по `source`.

2. **Смета не сформировалась, а статус уже «Согласована»**  
   Покрытие: генерация файла выполняется в том же транзакционном сценарии, с ошибкой возвращаем 500 и откатываем операцию.

3. **Расхождение файла сметы и данных заявки после редактирования**  
   Покрытие: пересборка сметы при каждом складском редактировании внешней заявки.

4. **Лишние кнопки/этапы в UI**  
   Покрытие: ограничение редактирования до старта сборки и сохранение условий действий в зависимости от статуса.

## Критерии готовности (Definition of Done)

- [ ] Внешняя заявка после создания сразу в статусе `APPROVED_BY_GREENWICH`.
- [ ] Внешняя заявка сразу имеет `estimateFileKey`, кнопка «Скачать смету» доступна.
- [ ] Редактирование внешней заявки до `PICKING` не откатывает статус в `SUBMITTED`.
- [ ] После редактирования внешней заявки смета обновляется.
- [ ] Внутренний Greenwich-флоу не изменён.
- [ ] Типизация и сборка проходят (`tsc`, `next build`).
