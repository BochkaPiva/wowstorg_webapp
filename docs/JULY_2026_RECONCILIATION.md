## Что сопоставлено

Шесть строк содержат прямые ссылки на существующие заявки/проекты WebApp:

- Клиника Фомина;
- Мега Фэшн Вик;
- Крылето55 04.07;
- Крылето55 11.07;
- Провокация 11.07;
- Авега ТурСлёт.

Две строки не имеют нормальной ссылки на сущность WebApp и ссылаются только на сетевые папки:

- `ДР_Алисы_2026`;
- `Додо_11.07.2026`.

Поэтому текущая аналитика сайта не может надёжно включить их как проектный факт без ручной идентификации.

## Обнаруженное расхождение

Ссылка/идентификатор, который ранее использовался для `ДР_Алисы_2026`, в WebApp указывает на другую по суммам заявку (`Диплей`). Автоматически заменять её финансовые данные нельзя: это может повредить складскую историю и расчёты уже существующей заявки.

## Безопасное решение

1. Создать для отсутствующих строк отдельные проекты/расчёты.
2. Внести смету как проектную версию, сохранив исходный Excel/сетевую папку в примечании.
3. После проверки владельцем перевести подтверждённый проект в `COMPLETED`.
4. Только затем включать его в финансовый факт и бонусный период.

Автоматическая запись данных из этой сверки в заявки, проекты и сметы не выполнялась.

## Реализованный контур сверки

В админ-аналитике добавлена вкладка «Сверка»:

1. администратор выбирает Excel и период;
2. интерфейс показывает предварительный просмотр без записи;
3. после явного подтверждения создаётся отдельный снимок;
4. для каждой строки отображается точное совпадение, конфликт или отсутствие сущности;
5. рядом выводятся итоги Excel, факт WebApp за тот же период и дельта.

Снимок не является финансовым фактом и не участвует в расчёте бонусов. Он нужен для диагностики и последующего ручного исправления первичных данных.
