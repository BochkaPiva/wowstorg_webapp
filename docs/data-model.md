# Data model (Prisma) — V1

Файл схемы: `prisma/schema.prisma`

## Цели модели
- Отразить роли (GREENWICH/WOWSTORG) и жизненный цикл заказа.
- Поддержать “цикл сметы” со snapshot’ами и возможностью повторного согласования.
- Поддержать критичный UX “приёмка по позициям” как разбиение количества по состояниям.
- Не завязывать дифф/сопоставление строк сметы на `orderLineId` (строки могут пересоздаваться).
- Сохранить историю цен: pricePerDaySnapshot + коэффициент payMultiplier.

## Основные сущности
- **User**: логин/пароль, роль, активность.
- **Customer**: справочник заказчиков.
- **Item**: позиция реквизита с типом и “ведрами” остатков (`total/inRepair/broken/missing`), флаг `internalOnly` для складских расходников.
- **Category + ItemCategory**: подборки/категории (многие-ко-многим).
- **Kit + KitLine**: пакет (набор) — хранится только как шаблон; при добавлении в корзину разворачивается в OrderLine.
- **Order**: заявка с датами (date-only), услугами, статусом, snapshot’ами сметы и согласования.
- **OrderLine**: строки заявки (item + qty), с опциональным `sourceKitId` и snapshot цены.
- **ReturnSplit**: универсальная таблица разбиения количества по состояниям для возврата:
  - `phase=DECLARED` — что указал Greenwich при отправке на приёмку
  - `phase=CHECKED_IN` — фактическая приёмка склада
- **Incident**: инциденты по негативным состояниям (создаются при check-in, если есть не-OK).
- **LossRecord**: недостачи с жизненным циклом OPEN → FOUND/WRITTEN_OFF.

## Важные инварианты (реализуются в коде, не только в БД)
- **date-only**: `readyByDate <= startDate < endDate`, нормализация к полуночи, ввод только `YYYY-MM-DD`.
- **Скидка Greenwich**: по умолчанию `payMultiplier = 0.76` для `GREENWICH_INTERNAL`, хранится на заказ.
- **Дифф сметы**: сравнение строк по `(itemId, sourceKitId)` с суммированием qty, а не по `orderLineId`.
- **Приёмка**: сумма `ReturnSplit.qty` по одной строке заказа и одной phase должна равняться “базовому количеству” (обычно `issuedQty ?? approvedQty ?? requestedQty`).
- **Транзакции**: все мутации статусов/выдачи/приёмки/складских ведер — внутри `prisma.$transaction`.
