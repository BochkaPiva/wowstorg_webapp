# Wowstorg Rental — V1 Scope (source of truth)

Дата: 2026-03-17

## Цель V1
Сайт для учёта аренды реквизита и взаимодействия **Greenwich ↔ Wowstorg(склад)**: маркет-каталог, заявки, согласование сметы, сборка/выдача, возврат/приёмка с инцидентами/недостачами, базовая аналитика.

## Роли
V1 фиксирует **2 роли**:

- **GREENWICH**
  - Видит каталог со скидкой.
  - Создаёт заявки из корзины.
  - Видит раздел **«Мои заявки»** (только свои).
  - Может редактировать/отменять заявку до начала сборки складом (см. статусы).
  - Получает смету и либо согласует, либо правит заявку и отправляет повторно на согласование.
  - После выдачи может отправить заявку на приёмку (быстро или по позициям).

- **WOWSTORG** (склад + инвентарь + админка)
  - Видит все заявки, очередь, архив.
  - Редактирует заявки, выставляет смету, запускает сборку, выдаёт, принимает возврат.
  - Управляет каталогом, подборками, пакетами, “внутренним инвентарём” (не в каталоге).
  - Ведёт списки: утеряно / требует ремонта / сломано (починить/найдено/списать).
  - Управляет пользователями и аналитикой.

## Два источника/типа заявок (OrderSource)
- **GREENWICH_INTERNAL**
  - Цена для пользователя — со скидкой **24%** (коэффициент оплаты 0.76).
  - Требует шага согласования сметы Greenwich.

- **WOWSTORG_EXTERNAL**
  - Цена — полная.
  - Согласование с заказчиком внутри системы не требуется (сразу идёт по складу).

## Скидка Greenwich (фикс)
Смысл: скидка 30% минус “возврат 6% на налоги” ⇒ итоговая скидка **24%**.

В расчётах V1 используем коэффициент:
- **payMultiplier = 0.76** (платят 76% от полной цены)

## Создание заявки (Greenwich) — обязательные/опциональные поля
После сборки корзины пользователь должен заполнить:

- **Обязательное**
  - Заказчик (Customer)
  - Даты (date-only):
    - `readyByDate` — к какой дате должно быть готово
    - `startDate` — начало мероприятия
    - `endDate` — окончание мероприятия
  - Состав (позиции + количество)

- **Опциональное**
  - Название мероприятия
  - Общий комментарий
  - Тумблеры доп. услуг:
    - доставка
    - монтаж
    - демонтаж
    - При включении каждой услуги появляется поле **«комментарий к услуге»** (свободный текст).

## Статусы заказа (OrderStatus) и разрешённые действия
Ниже приведены статусы V1 (названия в коде могут отличаться, логика — обязательна).

## Таблица действий по ролям (UI-кнопки)
Сокращения действий:
- **View**: просмотр деталей
- **Edit**: редактировать состав/кол-ва/комменты/услуги
- **Cancel**: отменить заявку
- **SendEstimate**: отправить смету (xlsx) + зафиксировать snapshot
- **ApproveEstimate**: согласовать смету
- **RequestChanges**: редактировать заявку и отправить “на согласование”
- **StartPicking**: начать сборку
- **Issue**: выдать
- **DeclareReturnQuick**: “Отправить на приёмку” (всё OK)
- **DeclareReturnByLine**: “Приёмка по позициям” (распределение состояний)
- **Checkin**: складская приёмка (факт)

| Status | GREENWICH | WOWSTORG |
|---|---|---|
| SUBMITTED | View, Edit, Cancel | View, Edit, SendEstimate |
| ESTIMATE_SENT | View, ApproveEstimate, Edit→RequestChanges, Cancel* | View, Edit, SendEstimate |
| CHANGES_REQUESTED | View, Edit, Cancel* | View, Edit, SendEstimate |
| APPROVED_BY_GREENWICH | View | View, Edit→SendEstimate, StartPicking |
| PICKING | View | View, Issue |
| ISSUED | View, DeclareReturnQuick, DeclareReturnByLine | View |
| RETURN_DECLARED | View | View, Checkin |
| CLOSED | View | View |
| CANCELLED | View | View |

`Cancel*`: отмена доступна только до `PICKING`.

### 1) SUBMITTED
Заявка создана Greenwich.
- GREENWICH: смотреть, **редактировать**, **отменить**
- WOWSTORG: смотреть, редактировать (состав/комменты/услуги/цены), подготовить смету

Событие: WOWSTORG получает уведомление о новой заявке.

### 2) ESTIMATE_SENT
Склад отправил смету (xlsx) и зафиксировал snapshot сметы.
- GREENWICH: смотреть смету, **согласовать**, либо **редактировать заявку** и “отправить на согласование”
- WOWSTORG: смотреть, при необходимости редактировать и отправить смету заново

Событие: GREENWICH получает уведомление + смету.

### 3) CHANGES_REQUESTED
Greenwich изменил заявку после сметы и отправил обратно “на согласование”.
- GREENWICH: смотреть (редактирование всё ещё возможно, пока склад не начал сборку)
- WOWSTORG: видеть обновлённый состав; желательно подсвечивать изменения относительно последней отправленной сметы

Событие: WOWSTORG получает уведомление об изменениях.

### 4) APPROVED_BY_GREENWICH
Greenwich согласовал смету.
- GREENWICH: смотреть (редактирование запрещено)
- WOWSTORG: может (а) начать сборку, (б) снова отредактировать и повторить цикл сметы

Важно: если склад внёс изменения после согласования, требуется повторная отправка сметы и новое согласование.

### 5) PICKING
Склад начал сборку.
- GREENWICH: только просмотр
- WOWSTORG: сборка → выдача

### 6) ISSUED
Заказ выдан/передан (склад ставит).
- GREENWICH: просмотр + действия по возврату (см. ниже)
- WOWSTORG: просмотр

### 7) RETURN_DECLARED
Greenwich отправил заявку на приёмку.
Есть 2 режима:
- **быстро**: “Отправить на приёмку” (всё ОК целиком)
- **по позициям**: распределение количества по статусам состояния + комментарии на негатив

В этом статусе GREENWICH **НЕ может** менять состав/количество позиций заказа — только распределение выданного количества по состояниям.

### 8) CLOSED
Склад завершил приёмку, складские остатки/ведра обновлены, инциденты/недостачи созданы/обновлены.

### CANCELLED
Отмена заявки.
Правило V1: GREENWICH может отменить только до `PICKING`.

## Приёмка “по позициям” (ключевой UX-инвариант)
Для каждой позиции заказа при возврате должен существовать набор строк состояния, суммарно равный количеству по позиции:
- `OK`
- `NEEDS_REPAIR`
- `BROKEN`
- `MISSING`

UI-инварианты:
- по умолчанию 100% количества в `OK`
- при выборе негативного статуса появляется поле “кол-во” и “хвост” автоматически остаётся в `OK`
- можно дробить дальше на несколько строк по разным статусам
- у каждой негативной строки есть обязательный/опциональный комментарий (решим в UX, но поле должно быть)

Складская приёмка:
- по умолчанию предзаполняется тем, что указал Greenwich
- склад может скорректировать статусы/кол-ва/комментарии перед закрытием

## Уведомления (канал уточняется)
События V1, на которые нужны уведомления:
- новая заявка (GREENWICH → WOWSTORG)
- смета отправлена (WOWSTORG → GREENWICH)
- изменения после сметы (GREENWICH → WOWSTORG)
- заявка согласована (GREENWICH → WOWSTORG)
- старт сборки/выдача (WOWSTORG → GREENWICH)
- возврат отправлен (GREENWICH → WOWSTORG)
- приёмка завершена/инциденты (WOWSTORG → GREENWICH, по необходимости)

Требование: уведомления **best-effort** — ошибки отправки не ломают бизнес-операции.

## Термины (словарь)
- **Смета / estimate**: расчёт стоимости по позициям + услугам + датам, формируется складом, отправляется как xlsx, фиксируется snapshot для сравнения.
- **Согласование / approve**: подтверждение сметы со стороны Greenwich (после этого Greenwich не редактирует заявку, пока склад не отправит новую смету).
- **Сборка / picking**: стадия, после которой редактирование заказа для Greenwich закрыто.
- **Приёмка / check-in**: фактическое принятие возврата складом с фиксацией состояний и созданием инцидентов/недостач.
